# MidGrid Harmonic Analysis Report Specification

This document specifies the text and JSON reports generated by `midgrid_parser.py`. The reports describe beat-aligned harmonic intervals, relative motion, and perceptual complexity between voice pairs.

## Outputs

For an output MIDI path such as `piece.mid`, the parser writes:

```text
piece.mid
piece.report.txt
piece.report.json
```

The text report is optimized for reading in prompts. The JSON report is optimized for automated repair loops and scoring.

## Text Format

The text report is a human-readable sequence of beat sections:

```text
Beat 0.00:
  V0-V1: interval=Perfect 4th [phase-aligned], motion=unknown, perceptual_complexity=7.0
  V0-V2: interval=Minor 6th [phase-aligned], motion=unknown, perceptual_complexity=13.0
```

### Section Header

- `Beat N.NN:` indicates the beat time of the analysis frame.

### Voice Pair Lines

Each pair line contains:

- `interval`: interval name, optionally annotated with octave displacement such as `(+2 oct)`.
- `[phase-aligned]`: present when the interval's harmonic relation is phase-aligned by the model.
- `motion`: `parallel`, `contrary`, `similar`, `oblique`, `unknown`, or `n/a`.
- `perceptual_complexity`: numeric interval complexity, or `n/a` when one or both voices rest.

## JSON Format

The JSON report uses schema `midgrid.report.v1`:

```json
{
  "schema": "midgrid.report.v1",
  "voice_count": 2,
  "beats": [
    {
      "beat": 0.0,
      "sounding_midis": [60, 52],
      "pairs": [
        {
          "voices": [0, 1],
          "voice_pair": "V0-V1",
          "midis": [60, 52],
          "interval": "Minor 6th",
          "interval_semitones": 8,
          "phase_aligned": true,
          "motion": "unknown",
          "perceptual_complexity": 13.0
        }
      ]
    }
  ],
  "summary": {
    "beat_count": 1,
    "active_pair_count": 1,
    "rest_pair_count": 0,
    "motion_counts": {"unknown": 1},
    "mean_perceptual_complexity": 13.0,
    "max_perceptual_complexity": 13.0,
    "max_perceptual_complexity_pair": {
      "beat": 0.0,
      "voice_pair": "V0-V1",
      "perceptual_complexity": 13.0,
      "interval": "Minor 6th"
    }
  }
}
```

### Beat Fields

- `beat`: numeric beat position.
- `sounding_midis`: active MIDI note per voice, or `null` for rest/silence.
- `pairs`: all voice-pair analyses for that beat.

### Pair Fields

- `voices`: numeric voice indices.
- `voice_pair`: stable string label such as `V0-V1`.
- `midis`: pair of MIDI notes, with `null` for rests.
- `interval`: interval name or `rest`.
- `interval_semitones`: absolute MIDI semitone distance, or `null` for rests.
- `phase_aligned`: boolean for sounding pairs, `null` for rests.
- `motion`: relative motion from the previous beat.
- `perceptual_complexity`: numeric score, or `null` for rests.

## Notes

- Voice indices follow the MidGrid voice-column order.
- Hold cells (`-`) are resolved as sustained sounding notes in `sounding_midis` and pair analysis.
- Motion analysis is best-effort and depends on the previous beat's sounding notes.
- The complexity score is a heuristic intended for comparison, not a complete aesthetic judgment.
- For automated composition loops, prefer JSON. For LLM prompt context, include short text excerpts plus JSON summary fields.
